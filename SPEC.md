# Kazan Mirage — Спецификация проекта

Интерактивный исторический атлас Казани. Пользователь открывает браузер и видит карту города с маркерами на ключевых достопримечательностях. Можно выбрать историческую эпоху — и описания мест, фотографии и панорамы меняются. Отдельный режим — татарская кухня: меню с блюдами, галереей фотографий и картой ресторанов.

---

## Технологии

| Что | Зачем |
|-----|-------|
| **Next.js 14** (App Router) | Фреймворк для React-приложения |
| **TypeScript** | Типизация всего кода |
| **Tailwind CSS** | Стили через классы прямо в JSX |
| **MapLibre GL JS** + `react-map-gl` | Интерактивная карта (тайлы OpenFreeMap, без API-ключа) |
| **Pannellum** (CDN) | 360°-просмотр панорамных фотографий |

## Данные (`data/`)

### `eras.json` — исторические эпохи

Массив из 4 объектов. Каждая эпоха:

```ts
{
  id: string        // "khanate" | "imperial" | "soviet" | "modern"
  label: string     // Название на японском, напр. "カザン・ハン国"
  years: string     // Годы, напр. "1438–1552"
  description: string  // Краткое описание эпохи
  mapStyle: string  // (поле зарезервировано, сейчас не используется)
}
```

### `places.json` — достопримечательности

Массив из 5 мест:
- Казанский Кремль (`kremlin`)
- Улица Баумана (`bauman`)
- Дворец земледельцев (`palace_agriculture`)
- Семейный центр «Казань» / «Чаша» (`chasha`)
- Озеро Кабан (`lake_kaban`)

Каждое место:

```ts
{
  id: string
  name: string            // Название на японском
  lat, lng: number        // Координаты для маркера на карте
  shortDescription: string
  coverImage?: string     // URL обложки (Wikimedia)
  eras: {
    khanate?: { panoramaImage: string, description: string }
    imperial?: { ... }
    soviet?:   { ... }
    modern?:   { ... }
  }
}
```

Ключ в `eras` — id эпохи. Если ключа нет — данных по этой эпохе не существует, кнопка эпохи в боковой панели будет недоступна.

### `restaurants.json` — рестораны

10 вымышленных ресторанов. Каждый:

```ts
{
  id: string
  name: string
  lat, lng: number
  address: string
  dishIds: string[]  // Какие блюда есть в меню (ссылки на dishes.json)
}
```

### `dishes.json` — блюда татарской кухни

10 блюд (щурпа, тутырган тавык, чак-чак и т.д.). Каждое:

```ts
{
  id: string
  name: string
  category: string          // "スープ" | "ペストリー" | "スイーツ" | "メイン料理"
  emoji: string
  shortDescription: string
  description: string       // Полное описание
  ingredients: string[]
  coverImage?: string       // URL главной фотографии
  gallery?: string[]        // Массив URL для галереи
}
```

## Стили

### `tailwind.config.ts`

Расширяет цветовую палитру Tailwind пятью именованными цветами:

| Имя класса | HEX | Где используется |
|------------|-----|-----------------|
| `parchment` | `#f5f0e8` | Основной цвет текста |
| `ink` | `#2c2416` | Текст на золотом фоне |
| `gold` | `#c9a84c` | Активные кнопки, акценты |
| `panel-bg` | `#1a1510` | Фон боковых панелей |
| `panel-border` | `#3a2e1e` | Разделители и рамки |

Классы используются во всех компонентах: `bg-panel-bg`, `text-gold`, `border-panel-border` и т.д.

### `app/globals.css`

Содержит стили, которые не покрываются Tailwind:

- **Сброс**: `html, body { overflow: hidden; height: 100%; }` — чтобы карта заняла весь экран без скролла
- **MapLibre-переопределения** (`.maplibregl-ctrl-group`, `.kz-maplibre-popup`) — перекрашивает стандартные элементы карты в историческую тёмную палитру
- **`.kz-pin`** — CSS-класс золотого маркера-ромба на карте (ромб = `border-radius: 50% 50% 50% 0` + `rotate(-45deg)`)
- **`.kz-pin--active`** — увеличенный и подсвеченный маркер для выбранного места
- **`.kz-food-pin`** — круглый красный маркер ресторана
- **`.kz-tooltip`** — стиль всплывающей подсказки при наведении на маркер

## Компоненты (`components/`)

### `KazanMap.tsx` — интерактивная карта

**Что делает:** рендерит MapLibre-карту с маркерами мест и ресторанов.

**Пропсы:**

| Проп | Что передаётся |
|------|---------------|
| `places` | Список мест для отображения (пустой массив в режиме ресторанов) |
| `selectedPlace` | Выбранное место (маркер подсвечивается, карта летит к нему) |
| `selectedEraId` | Текущая эпоха (сейчас не используется в карте, но передаётся для совместимости) |
| `onMarkerClick` | Колбэк при клике на маркер места |
| `restaurants` | Список ресторанов для отображения |
| `selectedRestaurant` | Выбранный ресторан (маркер подсвечивается) |
| `onRestaurantClick` | Колбэк при клике на маркер ресторана |
| `flyTo` | `[lat, lng]` — координаты, к которым карта должна полететь |

**Как работает маркер места:**
```
<Marker>        ← компонент react-map-gl, позиционирует на карте
  <div .kz-pin>  ← React-div с onClick → onMarkerClick(place)
    <span .kz-pin__icon>  ← SVG-иконка конкретного места
```

Иконки для каждого места захардкожены в объекте `ICONS` внутри файла (SVG-строки).

**Всплывающая подсказка:** при наведении на маркер показывается `<Popup>` с названием и кратким описанием места.

### `SidePanel.tsx` — боковая панель

**Что делает:** выезжает справа при клике на маркер. Показывает информацию о месте в выбранную эпоху.

**Пропсы:**

| Проп | Что |
|------|-----|
| `place` | Объект места из `places.json` |
| `era` | Объект текущей эпохи |
| `eras` | Все 4 эпохи (для рендера переключателя) |
| `selectedEraId` | Id активной эпохи |
| `onEraChange` | Колбэк смены эпохи |
| `onClose` | Закрыть панель |
| `onViewPanorama` | Открыть полноэкранную панораму |

**Структура панели сверху вниз:**
1. **Фото-обложка** (если есть `coverImage`) с именем места поверх градиента
2. **Переключатель эпох** — 4 кнопки в ряд. Кнопка активной эпохи выделена золотом. Если данных по эпохе нет (`place.eras[id]` отсутствует) — кнопка серая и заблокирована
3. **Описание** — текст из `place.eras[selectedEraId].description`
4. **Контекст эпохи** — общее описание эпохи из `eras.json`
5. **Кнопка панорамы** — кнопка «360°パノラマを見る», активна только если `place.eras[selectedEraId].panoramaImage` существует

**Анимация:** при открытии выезжает из-за правого края (`translateX(100%)` → `translateX(0)`) через `requestAnimationFrame`.

---

### `PanoramaFullscreen.tsx` — полноэкранный 360°-просмотр

**Что делает:** полностью перекрывает экран, показывает панорамное фото в выбранной эпохе.

**Пропсы:** `place`, `eras`, `selectedEraId`, `onEraChange`, `onClose`

**Структура:**
- **Верхняя панель** — кнопка «Back to map» и название места/эпохи
- **Центр** — `<PanoramaViewer>` (занимает всё оставшееся пространство)
- **Нижняя полоса** — переключатель эпох (аналогично `SidePanel`, но горизонтальный)

Переключение эпохи здесь меняет тот же `selectedEraId` в `page.tsx`, поэтому при возврате к карте боковая панель тоже покажет выбранную эпоху.

---

### `PanoramaViewer.tsx` — вьювер Pannellum

**Что делает:** загружает библиотеку Pannellum с CDN и рендерит 360°-вьювер в переданный `<div>`.

**Как работает:**
1. При первом рендере — динамически добавляет `<script>` и `<link>` с CDN (только один раз за сессию)
2. `imageUrl` из `places.json` хранится **без расширения** (напр. `/panoramas/kremlin_modern`). Компонент пробует по очереди `.png` → `.jpg` → `.webp` → `.jpeg`, пока один из файлов не загрузится
3. Когда файл найден — инициализирует `window.pannellum.viewer()` с авторотацией `-1.5 deg/s`
4. При смене `imageUrl` — уничтожает старый вьювер и создаёт новый

Если файл не найден ни с одним расширением — показывает заглушку с декоративной «компасной розой».

---

### `MenuPanel.tsx` — панель татарской кухни

**Что делает:** полноэкранное меню с тремя состояниями-экранами:

**1. `list` — сетка блюд**
- Блюда сгруппированы по категориям: スープ, ペストリー, スイーツ, メイン料理
- Каждая карточка: фото, название, краткое описание, количество ресторанов с этим блюдом
- Клик → переход в `detail`

**2. `detail` — страница блюда**
- Галерея фотографий с навигацией (стрелки + точки + полоса миниатюр)
- Полное описание и список ингредиентов в тегах
- Кнопка «地図でレストランを見る» → переход в `map`

**3. `map` — список ресторанов на карте**
- Боковая панель (на десктопе — справа, на мобильном — снизу как «bottom sheet»)
- Список ресторанов, у которых есть выбранное блюдо
- Клик по ресторану → карта перелетает к нему, маркер подсвечивается

### `IllustratedMap.tsx` — нарисованная SVG-карта

**Что делает:** полностью нарисованная SVG-карта Казани, без внешних тайлов. 900×620 пикселей, 15 слоёв:
- Пергаментный фон с текстурой
- Река Казанка (синяя полоса сверху)
- Холм Кремля с крепостными стенами и башнями
- Парки и деревья
- Улицы (главная — улица Баумана)
- Квартал старотатарской слободы
- Озеро Кабан
- Силуэты зданий
- Подписи рек, районов, улиц
- Картуш с названием
- Компасная роза
- Декоративная рамка с ромбами по углам
- Интерактивные маркеры мест (SVG `<g>` с `onClick`)

Сейчас этот компонент **не подключён** к главной странице — он существует как отдельная заготовка.

## Главная страница (`app/page.tsx`)

**Единственная страница приложения.** Собирает все компоненты вместе и хранит глобальное состояние.

**Состояние:**

| Переменная | Тип | Что означает |
|-----------|-----|-------------|
| `mapMode` | `"atlas" \| "restaurants"` | Текущий режим приложения |
| `selectedPlace` | `Place \| null` | Выбранная достопримечательность |
| `panoramaOpen` | `boolean` | Открыт ли полноэкранный просмотр |
| `selectedEraId` | `string` | Id активной эпохи (по умолчанию `"modern"`) |
| `shownRestaurants` | `Restaurant[]` | Рестораны, отображаемые на карте |
| `activeRestaurant` | `Restaurant \| null` | Выбранный ресторан |
| `flyTo` | `[lat, lng] \| null` | Куда карта должна полететь |

**Логика рендера:**

```
Если panoramaOpen && selectedPlace
  → показать <PanoramaFullscreen> (занимает весь экран)

Иначе:
  <header>  ← всегда видна. Содержит логотип и переключатель режимов
  <KazanMap>  ← занимает весь экран под заголовком, всегда
  
  Если mapMode === "atlas" && selectedPlace:
    <SidePanel>  ← поверх карты справа
  
  Если mapMode === "restaurants":
    <MenuPanel>  ← поверх карты (полноэкранно или сайдбар)
```

**Переключение режима** (`switchMode`): сбрасывает `selectedPlace`, `panoramaOpen`, `shownRestaurants`, `activeRestaurant`.

## Что можно делать в приложении

1. **Атлас** (режим по умолчанию): кликнуть на маркер → читать историю места → переключать эпохи в боковой панели → открыть панораму 360° → переключать эпохи в панораме
2. **Рестораны**: переключить режим → выбрать блюдо из меню → почитать описание и посмотреть галерею → нажать «показать на карте» → кликнуть на ресторан
